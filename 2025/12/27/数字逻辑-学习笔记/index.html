Number Systems

Digital Number Systems (Radix-rrr System)

Decimal (十进制)

Binary (二进制):

Octal (八进制): 00, 05, 013.

Hexadecimal (十六进制): 0x0, 0x9A, 0xCB.

Complements 补码

Diminished Radix Complement (反码): (r−1)(r - 1)(r−1)​​’s Complement

  • For Decimal System, DRC of 121212 can be 878787, 987987987…
  • For Binary System, it’s just a bitwise not (按位取反).

Radix Complement (补码): rrr​’s Complement

  • For Decimal System, RC of 121212 can be 888888, 988988988​…
  • For Binary System, equals to DRC +1+ 1+1.

Subtraction with Complements: M−N=M+(RC of N)M - N = M + (\text{RC of }N)M−N=M+(RC of N)

  • M≥NM \ge NM≥N: 丢弃高位进位
  • M<NM < NM<N: 加上负号

Binary Codes

BCD Code:

  • e.g., Decimal 396396396 is represented by 0011 1001 01100011 ~ 1001 ~ 01100011 1001 0110​.

  • Addition: 大于 999 时多加个 666。

  • Subtraction: Use 101010​’s Complement

Gray Code:

  • 相邻数字只有一位二进制数的区别。

ASCII Code

Error-Detecting Code:

  • 为了检测数据通信和处理中的错误，有时会在 ASCII 字符中添加一个额外的位（本例中为第八位）来指示其奇偶校验。奇偶校验位（校验位）是消息中包含的额外位，用于使 1 的总数为偶数或奇数。

Binary Logic

AND: z=x⋅y=xyz = x \cdot y = xyz=x⋅y=xy.

OR: z=x+yz = x + yz=x+y.

NOT: x′=x‾x' = \overline{x}x′=x.

  • AND、OR、XOR、Equivalence 有交换律、结合律。

Boolean Algebra

Boolean Axioms and Theorems:

(x+y)(x+z)=xx+xy+xz+yz=x+xy+xz+yz=x(1+y+z)+yz=x+yz\begin{aligned} (x + y)(x + z) &= xx + xy + xz + yz \\ &= x + xy + xz + yz \\ &= x (1 + y + z) + yz \\ &= x + yz \end{aligned} (x+y)(x+z)​=xx+xy+xz+yz=x+xy+xz+yz=x(1+y+z)+yz=x+yz​

Boolean Functions:

  • e.g., A′B′C+A′BC+AB′A'B'C + A'BC +AB'A′B′C+A′BC+AB′ has 888 literals, 333 product terms; (A+B′+C)(A′+C)(A+B'+C)(A'+C)(A+B′+C)(A′+C) has 555 literals, 222 sum terms.
  • Simplication: Minimize literals.

Minterms and Maxterms:

  • Minterm: an AND term consists of all literals in their normal form or in their complement form. nnn variables can be combined to form 2n2^n2n minterm.
  • Maxterm: an OR term of single variables. 2n2^n2n maxterms, each is the complement of a minterm.

Canonical Forms:

  • Sum of Minterms (SOM): Sum of all 111 terms.
  • Product of Maxterms (POM): Product of all 000 terms.
  • Conversion Between SOM and POM: ∑(1,3,6,7)=∏(0,2,4,5)\sum(1, 3, 6, 7) = \prod(0, 2, 4, 5)∑(1,3,6,7)=∏(0,2,4,5).

Standard Forms: terms may have fewer literals than minterms / maxterms.

  • Sum of Products (SOP): F1(x,y,z)=y′+xy+x′yz′F_1(x, y, z) = y' + xy + x'yz'F1​(x,y,z)=y′+xy+x′yz′.
  • Product of Sums (POS): F2(x,y,z)=x(y′+z)(x′+y+z′)F_2(x, y, z) = x (y' + z) (x' + y + z')F2​(x,y,z)=x(y′+z)(x′+y+z′)​.
  • Standard forms are not unique.

Simplification

Karnaugh Map (K-Map, 卡诺图) (SOP Simplification):

  • Two-Variable Map: e.g., m1+m2+m3=A+Bm_1 + m_2 + m_3 = A + Bm1​+m2​+m3​=A+B.

  • Three-Variable Map: Minterms are arranged in the Gray-code sequence. e.g., m0+m2+m5+m7=AC+A′C′m_0 + m_2 + m_5 + m_7 = AC + A'C'm0​+m2​+m5​+m7​=AC+A′C′.

  • Four-Variable Map: e.g., ABCD+AB′C′D′+AB′C+AB=AB+AC+AD′ABCD + AB'C'D' + AB'C + AB = AB + AC + AD'ABCD+AB′C′D′+AB′C+AB=AB+AC+AD′.

POS Simplification: Simplified SOP -> De Morgan’s Theorem -> Simplified POS

Don’t Care Conditions: 不知道的项，是否在组内无关紧要。

Logic Circuits

Combinational Circuits 组合电路

输出在任何时刻都直接且仅由当前的输入组合决定的逻辑电路。

Analysis Procedure:

  1. Derive the truth table.
  2. Determine the function, e.g., using K-Map.
  3. Explain if necessary.

Design Procedure:

  1. Specification: Determine inputs, outputs, and their symbols.
  2. Formulation: Derive the truth table.
  3. Optimization: Derive the simplfied boolean function for each output.
  4. Logic Diagram (optional)

Sequential Circuits 时序电路

Two Categories:

  • Synchronous Sequential Circuits (同步时序电路): 电路中有一个全局的时钟信号（Clock, clk），它像节拍器一样，产生一连串规律的脉冲。 所有的存储元件（比如“触发器” Flip-Flop）都连接到这个时钟上。只有在时钟的特定边缘（通常是上升沿（posedge），即信号从0跳到1的瞬间），触发器才会“睁开眼睛”看一下输入信号，并更新自己的状态。 在时钟的其他时间里，即使输入信号变来变去，触发器也“视而不见”，保持原来的状态不变。
  • Asynchronous Sequential Circuits (异步时序电路): 没时钟。

Storage Elements:

  • SR Latch: Asynchronous Sequential Circuit.

    • Clocked SR Latch:

  • D Latch / Transparent Latch: Asynchronous Sequential Circuit.

  • D Flip-Flop (DFF): Q(t+1)=DQ(t + 1) = DQ(t+1)=D.

  • J-K Flip-Flop: Q(t+1)=JQ′+K′QQ(t + 1) = JQ' + K'QQ(t+1)=JQ′+K′Q.

  • T Flip-Flop: Q(t+1)=T⊕QQ(t + 1) = T \oplus QQ(t+1)=T⊕Q.

Triggers:

  • Level-Triggered: Latches
  • Edge-Triggered: Flip-Flops

Analysis Procedure:

  1. Derive input equations for FF inputs
  2. Derive state and output equations
  3. Generate state and output tables
  4. Generate state diagram

Models:

  • Mealy Model: Output depends on both the state and the input.

  • Moore Model: Output depends on only the state.

FSM State Minimization:

  • For an FSM with mmm states, we need log⁡2m\log_2 mlog2​m FFs.
  • Reduction: repeatedly remove one of two equivalent states (who share the same next state and output).

Two Level Implementation

NAND & NOR

何逻辑函数或逻辑门（AND, OR, NOT）都可以仅用 NAND 门或仅用 NOR 门来实现。

SOP -> NAND-NAND Circuit

POS -> NOR-NOR Circuit

XOR & XNOR

Application: Parity Checking:

  • Generator: 利用 XOR 生成校验位 PPP。（偶校验）
  • Checker: 利用 XOR 检查数据位和校验位，C=0C = 0C=0 表示无错（或偶数个错），C=1C = 1C=1 表示有错。

Other Two Level Inplementations

AND-OR-Invert (AOI): F′F'F′ 的 SOP -> 取反

OR-AND-Invert (OAI): F′F'F′ 的 POS -> 取反

Arithmetic Circuit

Binary Adder

Half-adder: 两个 1 位二进制数相加的电路，输出和（Sum）与进位（Carry），不考虑低位进位。

Full-adder: 考虑有低位进位，相当于三个 1 位二进制数相加的电路。可用两个 Half-adder 实现。

Ripple-Carry Adder: 将多个全加器串联实现多位加法。主要缺点是延迟会随着位数线性增加。

Carry Lookahead Adder (CLA): 为了解决行波进位延迟问题，引入了进位生成（Generate）和进位传播（Propagate）的概念，通过并行计算进位信号来提高速度。

Binary Adder/Subtractor

Use 2’s complement. A−B=A+B′+1A - B = A + B' + 1A−B=A+B′+1.

Overflow:（有符号）V=Cn⊕Cn−1V = C_n \oplus C_{n - 1}V=Cn​⊕Cn−1​​ 时发生了溢出。

Serial Adder: 利用两个 Shift Register 和一个 Full Adder，逐位（从 LSB 到 MSB）进行串行相加的电路结构和时序。

Decimal Adder (BCD)

进位加 6。

Binary Multiplier

类似于手工乘法，通过生成 Partial Product 并将它们移位后相加来实现。

Other Arithmetic Functions

Design by Contraction: 通过将某些输入固定为 0 或 1，简化通用算术模块来实现特定功能（如将加法器简化为增量器）。

Incrementer / Decrementer: 加 1 / 减 1。

Shift Left / Right: 乘以 / 除以 2n2^n2n。

Zero/Sign Extension: 将低位宽数据转换为高位宽数据（无符号数补 0，有符号数复制符号位）。
